# Test docs

A suggested home for input documents you'll feed to `/doc-to-scenarios`
(test plans, requirements docs, meeting notes, acceptance criteria, etc.).

The plugin does not enforce this location — `/doc-to-scenarios` accepts
any path. Using `src/test/docs/` is just a tidy convention that keeps
input docs adjacent to the scenarios and tests they generate.

Delete this placeholder once you have real docs to put here.
